# v0.2.0 — Sprint 2 Model Comparison

## Release

`v0.2.0-sprint-02-model-comparison`

Status: **RELEASED — visual capture remains a documented limitation**

## Product capability

This candidate compares four classical regression approaches under one fixed
chronological experiment, documents their errors and limitations, separates the
metric leader from the integration candidate, and exposes the decision through
a versioned FastAPI resource and React dashboard.

## Included

- training-mean baseline, Linear Regression, Random Forest and Gradient Boosting;
- common MAE, RMSE and contextual R² contract;
- comparison table, prediction-level error analysis and selection policy;
- four Model Cards and three Decision Cards;
- `GET /api/v1/model-comparisons/summary`;
- loading, connected and unavailable frontend states;
- shared platform shell, safe logs, cross-layer checks and live HTTP smoke;
- final architecture, traceability, quality and portfolio documents.

## Verified evidence

- Gradient Boosting MAE: 3.0884 units;
- Random Forest MAE: 3.1258 units;
- practical-equivalence tolerance: 0.25 units;
- 96 Python tests, 18 frontend tests and 51 manual checks pass.

## Known limitations

- 18 synthetic observations and a six-row chronological holdout;
- no cross-validation, tuning, stability evidence or real-retail validation;
- no deployment, authentication or inventory action;
- no production-readiness claim;
- responsive screenshots are blocked by the current local-browser policy.
