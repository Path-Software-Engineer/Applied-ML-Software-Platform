# v0.1.0 — Sprint 1 Demand Insight

## Release

`v0.1.0-sprint-01-demand-insight`

## Product capability

This release turns a controlled retail-sales CSV into a user-facing Demand
Insight module. A retail analyst can review observed totals, leaders, baseline
error, contextual Insight Cards and analytical figures through a React dashboard
backed by a versioned FastAPI contract.

## Included

- validated loading, cleaning and integrated feature pipeline;
- temporal features and revenue;
- mean baseline 16.28 and MAE 5.42;
- sales, product and temporal summaries;
- five structured Insight Cards;
- three analytical PNG figures;
- `GET /api/v1/demand-insights/summary`;
- three allowlisted figure resources;
- responsive React Demand Insight dashboard;
- cross-layer tests, checks, architecture and sprint evidence.

## Verified analytical evidence

- 293 observed units and 747.65 observed revenue;
- Bread leads observed units with 105;
- Rice 1kg leads observed revenue with 220.50;
- 2026-01-06 leads daily units with 45;
- 2026-01-08 leads daily revenue with 99.30;
- mean baseline 16.28 and MAE 5.42.

## Validation

- 47 Python tests passed;
- 7 frontend contract tests passed;
- 23 manual checks passed at release readiness;
- deterministic frontend compilation passed;
- Vite production build passed;
- `git diff --check` passed.

## Known limitations

- The source contains 18 records across nine observed days.
- Results describe historical observations and are not forecasts.
- Currency is not inferred from the source contract.
- Model comparison and inventory decisions belong to later sprints.
- No deployment is included in this release.
