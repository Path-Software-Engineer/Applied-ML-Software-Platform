# 🧩 Path Software Engineer Roadmap — Plan 1

## 🧠 Applied ML Software Platform

**Applied ML Software Platform** es el primer plan de **Path Software Engineer**.

Este plan convierte la base técnica de Machine Learning clásico de **Path AI Engineer** en una aplicación de software real, robusta, documentada y orientada a producto.

El objetivo no es construir scripts aislados.

El objetivo es construir una plataforma aplicada que integre:

```txt
datos
→ análisis
→ baseline
→ métricas
→ modelos
→ decisiones
→ API
→ dashboard
→ documentación
→ evidencia profesional
```

---

# 🎯 Objetivo del plan

Construir una aplicación de software aplicada a inteligencia artificial para el dominio retail.

La aplicación debe permitir:

- analizar ventas;
- generar insights de demanda;
- comparar modelos clásicos;
- evaluar métricas;
- apoyar decisiones simples de inventario;
- mostrar resultados en un dashboard;
- documentar historias, decisiones y evidencias.

Este plan une fundamentos de software con Machine Learning aplicado.

# 📦 Proyecto del plan

Este plan contiene un proyecto principal:

```txt
01-retail-intelligence-platform
```

## 🧠 Proyecto 01 — Retail Intelligence Platform

**Retail Intelligence Platform** es una aplicación de software aplicada a IA para analizar ventas, comparar modelos clásicos y apoyar decisiones simples de inventario.

El proyecto integra tres módulos principales:

```txt
Sprint 1 — Demand Insight Module
Sprint 2 — Model Comparison Module
Sprint 3 — Inventory Decision Module
```

Cada sprint corresponde a un antiguo Building Project, ahora convertido en parte de una sola plataforma robusta.

Presenta el dashboard visual.

Guarda pruebas mínimas por capa.

Guarda comandos repetibles.

Guarda notas y archivos de despliegue.

# 🏃 Sprints del proyecto

## 🚀 Sprint 1 — Demand Insight Module

### Objetivo

Construir el primer módulo de la plataforma para analizar ventas, calcular métricas básicas y generar insights de demanda.

### Flujo

```txt
CSV local
→ carga
→ limpieza
→ features
→ baseline
→ MAE
→ análisis
→ insight cards
→ gráficos
→ dashboard inicial
```

### Resultado esperado

- total de unidades vendidas;
- revenue total;
- producto más vendido;
- producto con mayor revenue;
- mejor día de ventas;
- baseline promedio;
- MAE del baseline;
- insight cards;
- gráficos básicos;
- dashboard inicial.

---

## 📊 Sprint 2 — Model Comparison Module

### Objetivo

Agregar un módulo para comparar modelos clásicos de Machine Learning y convertir resultados técnicos en decisiones visuales.

### Flujo

```txt
dataset preparado
→ train/test split
→ baseline
→ modelos clásicos
→ métricas
→ tabla comparativa
→ error notes
→ decision cards
→ visual report
```

### Resultado esperado

- baseline vs modelos;
- tabla de métricas;
- modelo recomendado;
- notas de error;
- tarjetas de decisión;
- visual report;
- dashboard actualizado.

---

## 📦 Sprint 3 — Inventory Decision Module

### Objetivo

Agregar un módulo de decisiones simples de inventario usando demanda observada, stock actual, regla de reposición y tarjetas de riesgo.

### Flujo

```txt
ventas / inventario
→ demanda observada
→ stock actual
→ reorder rule
→ risk score
→ recommendation cards
→ inventory dashboard
```

### Resultado esperado

- demanda por producto;
- stock actual;
- productos con stock bajo;
- productos en riesgo;
- reorder point simple;
- risk score;
- recommendation cards;
- dashboard de inventario.
